# Bible Study Management Dashboard — CLAUDE.md

## Project overview

A **client-side** Next.js app to manage a Bible study promotion (2026–2027). No backend, no database — all data lives in `localStorage` under the key `bsm:data`. Four pages: Dashboard, Students, Minutes & Meetings, Settings & Import.

## Tech stack

| Layer | Choice |
|---|---|
| Framework | Next.js 16.2.6 (App Router, React 19) |
| Language | TypeScript 5.7.3 (strict) |
| Styling | Tailwind CSS v4 + shadcn/ui (base-nova) |
| UI primitives | Base UI React 1.5.0 |
| Icons | Lucide React |
| Toasts | Sonner |
| Persistence | localStorage (no backend) |
| Package manager | **pnpm** |

## Running locally

```bash
pnpm dev      # http://localhost:3000
pnpm build
pnpm lint
```

Default login: `admin / admin`

## Design language

- **Colors**: Blue (`oklch(0.55 0.21 256)`) + white. Hue 256 is the brand blue used throughout.
- **Style**: Minimal, professional. No heavy shadows, no gradients. Clean borders, soft backgrounds.
- **Cards**: `rounded-xl border border-border bg-card` — consistent border-radius everywhere.
- **Primary actions**: `bg-primary text-primary-foreground` (blue).
- **No dark mode** in the UI preference so far (CSS vars exist for it, but theme toggle is not wired up).

## Data model (lib/types.ts)

```ts
Student       { id, firstName, lastName, email, phone, birthday, classId, status, notes }
BibleClass    { id, name, order, description }
Lesson        { id, classId, title, order, description, reference }
CalendarEvent { id, date, type, title, description, lessonId?, classId?, studentId?, reference?, edited? }
Meeting       { id, title, date, description, createdAt }
Todo          { id, text, done, createdAt, meetingId? }
Settings      { promotionName, scheduleStartDate }
AppData       { settings, students, classes, lessons, events, meetings, todos }
```

`CalendarEventType = "lesson" | "fellowship" | "crusade" | "birthday"`

## Calendar generation rules (lib/scheduler.ts)

- One lesson every **Friday** and one every **Saturday**, in `class.order` → `lesson.order` sequence.
- A **Fellowship** every **Sunday** across the entire schedule span.
- A **2-day Crusade** (Fri + Sat) is inserted whenever a class finishes before the next class begins.
- **Birthdays** from students are added for every year covered by the span.
- Manually edited events (`edited: true`) are preserved by `${date}|${type}` key on regenerate.
- Mutations that change the schedule (settings, students, importData) call `mutateAndRegenerate`.
- `updateEvent()` does NOT regenerate — preserves manual edits.

## Store API (lib/store.tsx)

```ts
useStore() → {
  data, ready,
  updateSettings, regenerate,
  addStudent, updateStudent, deleteStudent,
  updateEvent,
  addMeeting, updateMeeting, deleteMeeting,
  addTodo, toggleTodo, updateTodo, deleteTodo,
  importData, exportData, loadSeed, resetAll,
}
```

## Event styling (lib/event-style.ts)

`EVENT_META[type]` → `{ label, icon, dot, chip, badge }` — import this to style any event consistently.

## Coding conventions

- All interactive pages: `"use client"` at the top.
- ISO dates everywhere: `yyyy-mm-dd`. Use `fromISO` / `toISO` from `scheduler.ts`.
- IDs generated by `uid(prefix)` in store — never generate IDs outside the store.
- Use `cn()` from `@/lib/utils` for conditional Tailwind classes.
- No comments unless the WHY is non-obvious.
- No extra abstractions — one file per route page.
- Stat cards pattern: `rounded-xl border border-border bg-card p-4` with a `bg-primary/10 text-primary` icon container.
- Tables: use the shadcn `Table` component with `bg-muted/40` header rows.
- Toasts: always use `toast.success()` / `toast.error()` from `sonner`.

## JSON import format

```json
{
  "students": [{ "firstName", "lastName", "email", "phone", "birthday", "classId", "status", "notes" }],
  "classes":  [{ "id", "name", "order", "description" }],
  "lessons":  [{ "id", "classId", "title", "order", "description", "reference" }]
}
```

All `id` fields in `students` are optional (generated if missing). `classId` in students must match a class `id`.
